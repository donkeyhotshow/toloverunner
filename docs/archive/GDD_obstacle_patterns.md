# Геймдизайн документ
## Геймплейные паттерны и системная логика препятствий

> **Версия:** 1.0 | **Дата:** 2026-02-27 | **Статус:** Draft

---

## 1. Дизайн-цели системы препятствий

**Задачи:**

1. Формировать обучающую кривую через повторяемые, но усложняющиеся паттерны.
2. Поддерживать ритм: напряжение → разрядка → награда.
3. Исключить «нечитаемую смерть» (смерть без телеграфа ≥0.35 с).
4. Обеспечить контролируемую плотность угроз (Threat Density Index).

---

## 2. Базовые метрики баланса

### 2.1. Core Speed Model

| Параметр              | Значение              |
| --------------------- | --------------------- |
| Базовая скорость      | 8 м/с                 |
| Рост скорости         | +0.15 м/с каждые 30 с |
| Максимум (стандарт)   | 14 м/с                |
| Хардкор                | до 18 м/с             |

### 2.2. Reaction Budget (минимальное окно реакции)

| Сложность | Окно реакции |
| --------- | ------------ |
| Легко     | 900–1100 мс  |
| Стандарт  | 650–850 мс   |
| Хард      | 450–650 мс   |

> **Правило:** Любой спавн обязан учитывать текущую скорость + окно реакции.

| -------------- | -------------------- | ------------------ | ---------------- |
| Статичная      | Низкая               | Низкая             | Высокая          |

**Запрещено:**
- Комбинировать 2 высоконагруженных паттерна в первые 90 сек.
- Спавнить босса во время активного Jetpack.

## 5. Паттерны уровня (структурно)

### 5.1. Паттерн «Обучение»
```
Статичная угроза → Одиночный тайминг → Комбинация 2 типов → Награда
```
Используется первые **2 минуты**.

---

### 5.2. Паттерн «Коридор давления»

- 3–5 движущихся угроз
- Ширина безопасной зоны: **≥1 трек**
- Продолжительность: **4–6 с**

---

### 5.3. Паттерн «Вертикальный риск»
```
Пандус → Бонусный верх → 2 траекторные угрозы → Jetpack или возврат
```
**Цель:** Проверка решения «риск/награда».

---

### 5.4. Паттерн «Когнитивный разрыв»

- Смена типа угрозы каждые **2–3 сек**
- Используется после **3-й минуты**

---

## 6. Threat Density Index (TDI)

**Формула:**

```
TDI = (Активные угрозы × Скорость) / Реакционное окно
```

**Целевые значения:**

| Сложность | TDI      |
| --------- | -------- |
| Легко     | ≤ 18     |
| Стандарт  | 18–26    |
| Хард      | 26–35    |

> ⚠️ Если TDI > 35 → вероятна фрустрация.

---

## 7. Процедурная генерация

### 7.1. Шаблонная архитектура

Секция = **8–12 секунд**

```
[Вход: разогрев] → [Основной вызов] → [Выход: наградной]
```

### 7.2. Ограничения генератора

1. Не более **1 фатальной угрозы** подряд.
2. После секции высокой плотности — «дыхательная» зона **≥3 с**.
3. Минимальный интервал между боссами — **90 с**.

## 9. Баланс усилений

| Усиление | Влияние на TDI        | Частота         |
| -------- | --------------------- | --------------- |
| Jetpack  | Обнуляет наземный TDI | 1 раз / 60–90 с |
| Dash     | −40% TDI на 0.4 с     | КД 6–8 с        |
| Shield   | Игнор 1 фатала        | Редкий          |
| Magnet   | Не влияет на TDI      | Средний         |

---

## 10. Ошибки, которых следует избегать

1. Случайная смерть без телеграфа.
2. Однообразный ритм >30 сек.
3. Чрезмерная зависимость от Shield.
4. «Фейковые» безопасные маршруты (монеты ведут в смерть).
5. Спавн препятствий вне поля зрения.
