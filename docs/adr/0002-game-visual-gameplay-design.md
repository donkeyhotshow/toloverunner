# ADR-0002: Дизайн визуала и геймплея ToLoveRunner

**Статус:** Принято  
**Дата:** 2026-04-04  
**Авторы:** AI Development Assistant (Tech Lead)  
**Версия проекта:** v2.4.0 → v3.0.0

---

## Контекст

ToLoveRunner — это 3D endless runner, где игрок управляет сперматозоидом, прокладывающим путь через биологические среды человеческого тела. Игра объединяет образовательно-юмористический нарратив с AAA-уровнем производительности (60 FPS stable, <50 draw calls). Требуется зафиксировать принятые решения по визуалу, геймплею и UX-дизайну для обеспечения согласованности при дальнейшей разработке.

---

## Проблема

При расширении команды или добавлении новых фич отсутствие единого ADR по дизайну приводит к:
- Нарушению палитры (внедрение «неоновых» цветов вместо «органических матовых»)
- Раздутию draw calls из-за неправильного выбора методов рендеринга
- Несогласованному UX между экранами (меню, HUD, Game Over)
- Потере нарративной идентичности (биологическая тематика)

---

## Решение № 1 (ПРИНЯТО): Органическая матовая палитра + Comic Illustration стиль

### Обоснование
- Игра имитирует биологическую среду → яркие неоновые цвета создают когнитивный диссонанс
- Матовые тёплые тона (flesh, blood, bone) соответствуют художественному стилю «биологическая иллюстрация»
- Ограниченная палитра снижает число уникальных материалов → меньше draw calls

### Правила палитры (ОБЯЗАТЕЛЬНО)
| Категория | Hex |
|---|---|
| Основной (плоть) | `#D2B48C` |
| Акцент (кровь) | `#8B2323` |
| Вторичный (желток) | `#DAA520` |
| Нейтральный (мышца) | `#8B4513` |
| Светлый (кожа) | `#DEB887` |
| Коральный (орган) | `#CD853F` |

**Запрещено:** использование чистого нейтрального синего (`#0000FF`), зелёного (`#00FF00`), любых neon/glow цветов в геометрии 3D объектов.  
**Допустимо:** мягкие эмиссивные эффекты для бонусов (не более opacity 0.4).

---

## Решение № 2 (ПРИНЯТО): Glassmorphism UI поверх органической 3D сцены

### Обоснование
- Glassmorphism создаёт чёткое визуальное разделение между 3D игровым миром и 2D HUD/интерфейсами
- `backdrop-filter: blur()` реализован на GPU — не нагружает CPU
- Полупрозрачность (opacity 0.15–0.25) сохраняет иммерсивность gameplay

### Правила UI
- Backgrounds HUD: `rgba(10, 0, 20, 0.75)` + `backdrop-filter: blur(12px)`
- Borders: `1px solid rgba(255, 255, 255, 0.08)`
- Text primary: `rgba(255, 255, 255, 0.95)`
- Text secondary: `rgba(255, 200, 150, 0.7)` (тёплый оттенок под тему)
- Кнопки: gradient `from rgba(139,35,35,0.8) to rgba(218,165,32,0.8)` + hover scale 1.05

## Решение № 3 (ПРИНЯТО): Five-lane система с procedural Pattern Generation

### Обоснование
- 5 полос (`-2, -1, 0, 1, 2` × LANE_WIDTH 2.0) дают баланс между хаосом и предсказуемостью
- Pattern Generator берёт паттерны из каталога (easy/medium/hard/expert) с учётом текущей скорости и навыка игрока (`playerSkill 0–1`) — это предсказуемее чем pure random
- Subway Surfers доказал жизнеспособность этой механики при добавлении вертикального измерения (прыжок/скольжение)

### Правила геймплея
- Базовая скорость: 10 m/s → потолок 45 m/s при score progression
- Прыжок: высота 2.2 единицы, дуга 0.5 сек (ease-out)
- Double jump: сила 12.5, падение с множителем 1.3 (вес при приземлении)
- Coyote time: 120 мс (прощающая механика для новичков)
- Jump buffer: 180 мс (для отзывчивости)

---

## Решение № 4 (ПРИНЯТО): Биомы как нарративные акты

### Обоснование
- Маршрут игры = биологический путь сперматозоида: фаллопиева труба → иммунная система → яйцо
- Каждый биом меняет цветовую схему, плотность тумана и тип врагов — создаёт ощущение прогрессии без смены механик
- Смена биома каждые ~500–800 метров удерживает интерес

### Матрица биомов → враги
| Биом | Ключевые враги |
|---|---|
| Fallopian Tube | Bacteria (Низкие барьеры) |
| Bio Jungle | Worms (Globus types) |
| Vein Tunnel | Immune Cells (Patrol/Aggressive) |
| Egg Zone | Viruses (смертельные) |
| Immune System | Boss patterns |

## Альтернативы, которые были отклонены

| Вариант | Причина отказа |
|---|---|
| Neon/Cyberpunk стиль | Противоречит биологической тематике |
| Pure random obstacle generation | Непредсказуемо, нарушает стабильность сложности |
| Плоский 2D HUD без glassmorphism | Теряет премиальное ощущение, выглядит дёшево |
| Одна полоса (Geometry Dash-like) | Слишком ограниченно для 3D |

---

## Последствия

- **Положительные:** Единая дизайн-система, меньше конфликтов при code review, быстрее onboarding новых разработчиков
- **Риски:** Glassmorphism `backdrop-filter` не поддерживается на старых мобильных GPU → необходим fallback (`background: rgba(10,0,20,0.92)`)
- **Мониторинг:** Draw calls должны оставаться < 55 (целевой 48); Memory growth < 10 KB/sec
