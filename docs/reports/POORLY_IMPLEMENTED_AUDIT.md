# Аудит: что реализовано плохо или неполно

Отчёт по результатам анализа кодовой базы (февраль 2026).

---

## 1. Критично: звуки от физики не воспроизводятся

**Проблема:** В `useGamePhysics.ts` при столкновении и сборе предметов вызывается:
```ts
window.dispatchEvent(new CustomEvent('play-sound', { detail: { sound: 'enemy-hit', volume: 0.8 } }));
```
Но **UnifiedAudioManager** подписан только на **eventBus** `'system:play-sound'`, а не на `window` событие `'play-sound'`. Моста между ними нет.

**Следствие:** Звуки `enemy-hit`, `gene-collect`, `coin-collect`, `powerup-collect` из физики **не играют**.

**Решение:** Либо в одном месте (например, в `App.tsx` или при инициализации аудио) добавить:
```ts
window.addEventListener('play-sound', (e: CustomEvent) => {
  eventBus.emit('system:play-sound', e.detail || {});
});
```
Либо заменить в `useGamePhysics` все `CustomEvent('play-sound', ...)` на `eventBus.emit('system:play-sound', ...)`.

## 2. Щит: при блоке всё равно проигрываются эффекты удара

**Проблема:** При столкновении с препятствием с активным щитом вызываются:
- `store.takeDamage()` (внутри щит корректно блокирует урон),
- но до этого уже отправляются `player-hit`, `particle-burst`, `play-sound` (enemy-hit).

**Следствие:** При блоке щитом игрок видит/слышит удар и частицы как при реальном попадании.

**Решение:** В `useGamePhysics` перед обработкой столкновения с препятствием проверять `store.shieldActive`. Если щит активен — не вызывать `takeDamage`, не слать `player-hit` / `particle-burst` / `play-sound` для удара; можно отправить отдельное событие `shield-block` и воспроизвести другой звук/эффект.

## 3. Магнит тянет спец-препятствия (Jump/Slide/Dodge)

**Проблема:** В цикле магнита в `useGamePhysics` пропускается только `ObjectType.OBSTACLE` (вирусы):
```ts
if (!obj || !obj.active || obj.type === ObjectType.OBSTACLE) continue;
```
Типы `OBSTACLE_JUMP`, `OBSTACLE_SLIDE`, `OBSTACLE_DODGE` не исключаются.

**Следствие:** Шипы, слайд-барьеры и стены визуально притягиваются к игроку магнитом, что выглядит как баг.

**Решение:** Исключать все опасные типы, например:
```ts
const isHarmful = obj.type === ObjectType.OBSTACLE || obj.type === ObjectType.OBSTACLE_JUMP ||
  obj.type === ObjectType.OBSTACLE_SLIDE || obj.type === ObjectType.OBSTACLE_DODGE;
if (!obj || !obj.active || isHarmful) continue;
```

## 4. Два игровых цикла, один не используется

**Проблема:** В проекте есть:
- **GameLoopRunner** — реальный цикл через `useFrame` (R3F), вызывает коллбэки из `GameLoopRegistry`;
- **OptimizedGameLoop** — свой `requestAnimationFrame` и `executeTasks()`, но в `GameLoopRunner` закомментировано `gameLoop.start()`, задачи в него не добавляются.

**Следствие:** Мёртвый код, путаница при чтении, лишний синглтон.

**Решение:** Либо удалить `OptimizedGameLoop` и связанные хелперы, либо явно использовать его (например, для фоновых задач) и зафиксировать это в архитектурном документе.

## 5. Интерполяция физики не используется для рендера

**Проблема:** **PhysicsStabilizer** считает интерполированное состояние (`getInterpolatedState()`), но в рендере позиция игрока берётся из `store.localPlayerState`, который обновляется из `finalPlayer.position` после шагов физики. Интерполяция стабилизатора нигде не подмешивается.

**Следствие:** При нестабильном FPS или при нескольких подшагах за кадр возможны дёргания, хотя стабилизатор мог бы их сгладить.

**Решение:** Либо в цепочке рендера игрока использовать `getPhysicsStabilizer().getInterpolatedState()` и выставлять позицию из неё, либо убрать/упростить интерполяцию в стабилизаторе и явно описать, что сглаживание не используется.

## 6. Детерминизм реплеев нарушен

**Проблема:** В `useChunkSystem` id объекта задаётся так:
```ts
const objectId = `obj_${safeDistId}_${i}_${Math.random().toString(36).substr(2, 5)}`;
```
Используется `Math.random()`, а не RNG из **ProceduralSystem**.

**Следствие:** При одном и том же сиде генерации чанков объекты будут с разными id при каждом запуске — реплеи/воспроизведение по сиду не будут строго детерминированными.

**Решение:** Либо передавать в чанк-коллбэк опциональный RNG/seed и генерировать id из него, либо формировать id только из `safeDistId` и `i` (без случайной части), если уникальность и так обеспечена.

## 7. Мелкие замечания

- **activateShield** в `gameplaySlice` выставляет `isImmortalityActive: true` на время щита — по смыслу это «неуязвимость», а не бессмертие; название может сбивать с толку при поддержке.
- **Опечатка в лицензии:** В `OptimizedGameLoop.ts` в первой строке указано `@licse` вместо `@license`.
- **Типизация:** В `useGamePhysics` используется `let lastCollision: any = null` — лучше задать тип `CollisionResult | null`.
