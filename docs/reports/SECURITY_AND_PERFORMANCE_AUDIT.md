# Аудит безопасности и быстродействия

**Дата:** 2026-02-22  
**Цель:** Сканирование на уязвимости и способы улучшить быстродействие (quick wins).

---

## 1. Безопасность

### 1.1 Проверено

| Область | Результат |
|--------|------------|
| **eval / new Function / innerHTML** | В `.tsx` и коде приложения не найдено. Опасного выполнения строки кода нет. |
| **dangerouslySetInnerHTML** | Не используется. |
| **Секреты в коде** | Sentry DSN — из `import.meta.env.VITE_SENTRY_DSN`. В `constants.ts` только публичные значения (PEER_KEY для PeerJS и т.п.). Хардкод секретов не обнаружен. |
| **localStorage** | Используется в SaveManager, GestureManager, I18nManager, ProgressionSystem, AccessibilityManager. Данные — настройки и сохранения; в разметку не подставляются. XSS через хранилище не выявлен. |
| **Импорт сохранений** | Раньше: неограниченный размер строки → риск DoS через `JSON.parse` на огромной строке. **Исправлено:** введён лимит `MAX_IMPORT_SIZE = 500_000` (500 KB) в `SaveManager.import()`. |

### 1.2 Рекомендации

- **npm audit:** выполнять периодически (`npm audit`) и при обновлении зависимостей; критичные уязвимости устранять в первую очередь.
- **Импорт:** при необходимости можно ужесточить лимит (например 200 KB) или вынести его в константу/конфиг.
- **Валидация сохранений:** структура проверяется через `validateSaveData`; при появлении полей из ненадёжных источников — проверять типы и диапазоны (числа, длина массивов/строк).

## 2. Быстродействие — внесённые изменения

### 2.1 VirusObstacles: аллокации в цикле

- **Проблема:** В цикле по шипам каждого вируса на каждый кадр создавались `new Vector3()` и `new Euler()`, что давало лишний мусор и нагрузку на GC.
- **Решение:** Добавлены переиспользуемые объекты в ref: `spikeVecRef`, `spikeEulerRef`. В цикле вызываются `.set()` и `.applyEuler()`, без новых аллокаций.
- **Файл:** `components/World/VirusObstacle.tsx`.

### 2.2 SaveManager: защита от DoS при импорте

- См. п. 1.1 — лимит размера строки перед `JSON.parse` снижает риск зависания при импорте огромного «бэкапа».

## 3. Быстродействие — рекомендации (quick wins)

| Приоритет | Что сделать | Где | Эффект |
|-----------|-------------|-----|--------|
| Высокий | Условно отключать SSAO и DoF на низком качестве | PostProcessing.tsx | Меньше нагрузка на слабых устройствах |
| Высокий | Убедиться, что при паузе вызывается `resetAccumulator()` у PhysicsStabilizer | GameLoopRunner / смена статуса | Уже сделано в предыдущей сессии |
| Средний | Единый флаг готовности сцены (gameSceneReady) | App, PostProcessing, RenderController | Меньше гонок и лишних рендеров |
| Средний | Аудит dispose/release геометрий и материалов при unmount | Компоненты с BufferGeometry/Material вне пула | Меньше утечек памяти при переходах между сценами |
| Низкий | Throttle/дебаунс частых обновлений UI (счёт, дистанция) | ComicTopBar, панели HUD | Меньше ре-рендеров React |
| Низкий | LOD по дистанции для декора/препятствий | BiomeDecorRenderer, InstancedLevelObjects | Меньше draw calls на дальних объектах при низком качестве |

Детали по движку, рендеру и стабильности — в `docs/IMPROVEMENTS_BACKLOG.md`.

## 4. Итог

- **Уязвимости:** опасных паттернов (eval, innerHTML, хардкод секретов) не найдено. Добавлен лимит размера импорта сохранений.
- **Быстродействие:** убраны аллокации в горячем цикле VirusObstacles; остальные шаги — в бэклоге с приоритетами.
- **Рекомендация:** регулярно запускать `npm audit` и при появлении новых зависимостей проверять их на известные CVE.
