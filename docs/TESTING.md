# 🧪 Руководство по тестированию

**Версия:** 2.1.0

---

## 🚀 Быстрый старт

### Запуск dev-сервера

```bash
cd toloverunner
npm run dev
```

Сервер запустится на `http://localhost:3000`

---

## 🔍 Мониторинг утечек памяти

### Активация мониторинга

Откройте консоль браузера (F12) и выполните:

```javascript
// Скопируйте и вставьте содержимое scripts/monitor-dev.js
// Или просто выполните:
fetch('/scripts/monitor-dev.js').then(r => r.text()).then(eval);
```

### Тест перезапуска игры

```javascript
// 1. Начальный snapshot
__TOLOVERUNNER_MONITOR.takeMemorySnapshot('start');

// 2. Запустите игру и играйте 30 секунд
// 3. Game Over → Restart
// 4. Повторите 5-10 раз

// 5. Финальный snapshot
__TOLOVERUNNER_MONITOR.takeMemorySnapshot('after_10_restarts');

// 6. Сравнение
__TOLOVERUNNER_MONITOR.compareSnapshots(0, -1);

// 7. Проверка интервалов
__TOLOVERUNNER_MONITOR.getIntervalStats();
// Ожидаемый результат: active < 5 (только системные)
```

## ⏱️ Тестирование Power-up таймеров

### Мониторинг таймеров

```javascript
// Запустите мониторинг
const stopMonitoring = __TOLOVERUNNER_MONITOR.monitorPowerUps();

// Соберите Shield power-up
// Наблюдайте в консоли

// Остановить мониторинг:
stopMonitoring();
```

### Тест повторной активации

1. Соберите Shield → таймер должен показать 10.0
2. **Сразу соберите еще один Shield** (если возможно)
3. Проверьте, что таймер не дублируется

---

## 📊 Ключевые метрики для проверки

| Метрика | Целевое значение | Как проверить |
|---------|------------------|---------------|
| Heap разница | < 5MB | `compareSnapshots()` после 10 перезапусков |
| Активных интервалов | < 5 | `getIntervalStats()` |
| FPS | ≥ 55 | DevTools Performance |
| Memory тренд | Стабильный | Memory Profiler |
| Power-up таймеры | Корректные | `monitorPowerUps()` |

## 🎯 Приоритеты тестирования

### Критично (сначала):
1. **Утечки памяти** - перезапуск игры 10 раз
2. **Power-up таймеры** - повторная активация
3. **Производительность** - длинный ран (5+ минут)

### Важно (затем):
4. Race conditions - быстрые переключения статусов
5. Физика - проверка NaN и корректности
6. Коллизии - граничные случаи

---

## 🛠️ Инструменты для тестирования

### 1. Автоматический мониторинг
- Скрипт: `scripts/monitor-dev.js`
- Автоматически загружается в dev-режиме
- Доступен через `window.__TOLOVERUNNER_MONITOR`

### 2. Экспорт store
- Доступен через `window.__TOLOVERUNNER_STORE__` (только в dev)
- Позволяет проверять состояние игры из консоли

### 3. 🔍 Диагностика рендера
- Скрипт: `tests/RenderDebug.js`
- Пошаговая диагностика draw calls = 0
- Доступен через `window.__RENDER_DEBUG__`
- Test Runner: `test-runner.html` → "🔍 Диагностика рендера"
